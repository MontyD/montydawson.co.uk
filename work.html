Todoist | Personal Project

In order to solidify my knowledge of Angular and ES6, I did what everyone does - created a todo list app. To push myself I decided to build in the ability to collaborate on todo lists. I wanted to create something where people could log in and act on todo lists together, with actions being updated across all users in real time. This was challenging, but something I ultimately pulled off. I added a few nice features on the frontend, including a fun "overview" page containing a graph of weekly progress. The backend is built on the ExpressJS framework and postgres for the database. The frontend uses Angular 1 and Sockets.io as a wrapper for the Websockets API.

Although it's still a work in progress I really like how it has come together. I've learnt a lot about tracking state and data in an asynchronous environment as well as greatly improving my JS skills. The site is hosted on the free tier on Heroku - so will take a few seconds to "wake up" when first called.

Dublove | Client project

A small site for a local VW chaufeeuring company. This site helped me grow my skills at developing for the PHP CMS Silverstripe. I took advantage of the rich API, creating a customisable theme, complete with front-end forms for customers to use to raise inquiries and leave feedback. I ensured that the site was as lightweight as possible, with all images being lazy-loaded, resources minified and gziped, and all vector graphics loaded through gruntify - serving the most effecient version supported by the browser. I also developed my skills at writing efficient modular JS, using webpack and grunt for bundling and minification.

I am very proud of this site, and have really enjoyed working with the client to put something together that is developed to their brief and coded cleanly and efficiently. The site is due to go live at the end of this year.

A website in under three minutes | Personal project

I had the fairly stupid idea that it would be fun to create a website that codes itself in real time - with the process of writing HTML, CSS and JS being shown to the user. So that's what I made. It was fun pushing my knowledge of JS - and again helped me in writing modular and efficient code. The end result is pretty good, with HTML CSS and JS being written and applied in "real time" by looping through a JSON consisting of snippets of code interlaced with comments. The syntax highlighting of the code is particularly nice - aided by the prism.js library.

The website made at the end of the three minutes is nothing special at all - but I am proud of the way that it's delivered with all the code to see.

The Well | Client Project

The idea was to create a simple website which contained the menu, opening hours, some photos, and a bit of information about the purpose that this Christian Coffee Shop serves. I took my time and did things from scratch, creating the CSS and JavaScript without any frameworks. Because the content was open to change and there was the change of adding more functionality later on, I decided to use the very flexible PHP CMS SilverStripe. This was the biggest challenge for me, and helped to develop my back-end skills.

I am very happy with the result, it draws on elements of The Well's branding and colours, it's fully responsive, the content is presented well, and the CMS is user-friendly so that anyone can make changes

SlackQuiz | Personal Project

My first go at solo web development produced this amazing peice of engineering. It started life as a chat bot for the Slack platform - you would say "Quizme new question", and it would give a question - then people could answer with "Quizme" followed by their answer. Then I developed a frontend, allowing people to add their own questions and a leadboard section with a nice bar chart. It was written in NodeJS before I understood how asynchronous programming worked, and the frontend was a complete mess of Jquery and regrets. Oh also it used MongoDB - because at the time I was too cool for SQL.

It was a very fun little app that provided a bit of a laugh in a Slack group. Although the code is a complete mess, it's nice to look back on and see how far I've come.
